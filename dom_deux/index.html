DOM EVENTS

The DOM

When our browser loads a web page, all the HTML elements are represented by the DOM. HTML elements go in...JS Objects come out. All of these objects have certain properties available. There is a relationship between these objects and how they are organised called a tree. This is a data structure. The top level object is called the document. The object has all of our properties and methods to help us manipulate the DOM! Use console.dir(document) to see it all!

Get Elements by tagName

This will return all elements with a specific tag name so all img or all p. In the console we will see a HTML collection. It looks like an array but it is not. We can iterate over the collection. What is inside the collection? An element.

Elements by Class

We can add a class name to each element and use the class to retrieve all elements with the same class. If there are no elements with a specific class then we get an empty collection.

This provides us with the first match. But if we want all the elements then we us document.querySelectorAll("#img"). When we use selectorAll we are given a NodeList. What is a NodeList? We can select all descendents just like when we are styling in CSS, this is because we use CSS selectors to pass into the querySelector method. Properties and Methods

TextContent, innerText and innerHTML

All of these properties are related to the text inside an element. We can grab any element in the dom and update the text with the following syntax:

  • document.querySelector('p').textContent = "new text"
  • document.querySelector('p').innerText = "new text"

The difference is that textContent recognises hidden elements set to display: none. Whereas innerText is sensitive to what is showing up at the moment.

InnerHTML is useful for updating elements to valid HTML, not possible with textContent or innerText.

Attributes

What are attributes?

There are so many attributes that live on elements. We can update them using JavaScript. We can get all the attributes on an element using getAttribute and change attributes using setAttribute. Be aware we are just accessing the html, we are not accessing the object in the DOM. Common attributes include:

    id
    class
    src
    href

Changing Styles

The styles object does not contain the current styles of an element if you define them in a separate style sheet. It only conatins inline styles which are the the best way to do things. We can use the style object to change styles. Here is an example: h1.style.color = "green". But we are writing inline styles using JavScript. There is a better way and this is to define a class! Since this style object is empty we can also inspect any elements style and this is using a special method called window.getComputedStyle("element to inpspect")

Traversing

We can use parentElement to find the parent of any element. We can also use children to access child elements. Siblings are elements that all exist on the same level. There is previous and next sibling and they give us the corresponding node. A node is different from an element. Nodes can represent text. This can be white space or new lines. A better way is get access to the sibling is nextElementSibling.

Creating Elements

We can use const newImg = document.createElement('img') to create an element. Once we have made an element it needs to be added to the DOM. So we use the append property. First we need to describe the place where the element will be added, append property and then the element to be added like so: document.body.appendChild(newImg). Once we have made the element we should add any propeties such as src before adding to the DOM. Prepend allows us to add an element as the first child. We can use insertAdjacentElement to go before or after an element.
