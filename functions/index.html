Functions

Functions are the building blocks of JavsScript!

Functions are reusable, modular blocks of code. We can execute them at a later date in time and we use them all the time! Examples include making a function to roll a die. Instead of writing out 6 variables we can create just one function and call it. We can extend the analogy to make a dice with an input that changes the sides of the die. This two step process is critical. You SHOULD define your functions before you call them.

Functions

Creating a function is a two step process. First we define a function and then we run or invoke it. This is how we declare a function: function funcName(){//do something}. Function names are camel cased. The code inside the curly braces will be run when the function is called. And this is how we invoke the function: functionName().

Arguments are just a fancy way of saying inputs to our function. For example we can print a custom message to the console based on the arguments we pass to the function. It refers to the value passed in. When we used methods like array.push(4) this is technically an argument being passed in to the push method. The parameters are a variable that represents the value passed in. If we are expected an argument and nothing is passed in when the function is called we will get undefined returned.

Multiple arguments

We can pass in multiple arguments to a function. The order matters, so in the example if we switched the order of the parameter the result would be reversed.

Return Key Word

The return keyword! So far we have been console logging, but this is not a value that we can reuse. We need to be able to store values in a variable or use a method on it. We should return a value from our function using the return keyword return. Return will stop the execution of the function.

Levelling up our functions

Scope

Scope is a challenging concept! So what exactly is scope? Scope refers to variable "visiability". The place where the variable is declared impacts how it can be accessed.

Function Scope

If a let variable is defined within a function, we cannot access it from the outer code. If we have a conflict and a variable with the same name exists outside the function, the function will ignore this outer variable.

Block Scope

Block refers to any code held in curly braces such as conditions or loops. var does not respect the block scope. let and const variables decalred in block only exist within that block!

Lexical Scope

Lexical Scope means that a function has access to the variables in the outer scope or parent function. So in the example below we can see that cryForHelp has access to the heroes variable.

Function Expressions

There is another way to create a function. That is a function expression. We store the function we have created in a variable. When we can the function by using the variable name. We are essentially craeting a function with no name but the variable has a name. Functions are values in JavaScript and we can pass them around and return a function just like a return value!

High Order Functions

A high order function is a function that can be passed around as an argument. For example in callTwice we are passing rollDie as an argument. We can also return a function as value from a function. The pattern her is the important thing to learn. Sometimes you will also see a function that generates another function based on some input! We can capture the return value in a variable. This pattern is known as a factory function!

What is a Method?

A method is a function that is a property on an object. If we inspect the Math object in the console we will see it made up of a bunch of properties. There is a shorthand properties and we can remove the function keyword.
