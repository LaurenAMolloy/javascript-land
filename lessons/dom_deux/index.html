DOM EVENTS

DOM events are the key to making interactive websites when a user does certain things like clicks, drags, drops, scrolls, keypresses, form submission or mouseover.

Events in JavaScript!

Inline Events Suck!

There are three ways to respond to user clicks. The first way is to do this is kinda janky! This involves adding an attribute called onClick and NOT recommended. It is not resuable and if we want the same functionality it is alot of copy and pasting. What if you have multiple buttons on the same page with the same behaviour? This approach will not work!

Another Way to Add Events

The second way we are going to add an event is to create a function that runs when an element is clicked. This is done purely using JavaScript. This makes the functionality easier to reuse. Any element can be made interactive, we are not limited to buttons.

Event Listeners

The best ways

The best way to add an event is to add an event listener. We do this by selecting the element with query selector and adding and event listener. This will allow us to listen to any event we want! Here is a list of events. Why is this better? The first reason is we can apply as many callbacks as we want. You may have more than one callback to run on a certain element. We can also pass in options object to add extra control. In complex apps we may need to remove eventlisteners using removeEventListener

    Event Bubbling!

    What is event bubbling? Event bubbling is when an event is triggered on an element. There may be other events triggered by this event. If we have nested elements they will be triggered each time. We can stop this bubbling by using e.stopPropagation()

    Event Delegation

    This is a strategy for writing code using events. We add our event listener to a parent element instead of the individual elements. We can still listen for specific clicks on child elements. We use the event object for this. We are interested in the target inside the event. We are going to look at the target and check if it is an li and if it is we can remove it.
