Decision Making

Boolean Logic

You should care about these concepts as they apply to any programming language. For example we can use boolean logic for multiple interactions with a website or app. If we like a post and it has a heart or thumbs up, that is a logical decision that we can make via our code.

Comparison Operators

We often need to compare values to make a decision in JavaScript. For example we can check if a value is greater or less than another and it will give us a boolean value(true or false). We will commonly use the following comparison operators:

  • Greater than >
  • Greater than or equal to >=
  • Less than <
  • Less than or equal to <=
  • Loosley equal to ==
  • Strictly equal to ===
  • Not equal to !=
  • Strictly not equal to !==

Triple vs Double Equals

This is often a source of confusion! If we compare a string of 1 and a number of 1 they will be considered the same. JavaScript will coerce the values to the same type. However triple equals does care about type and when we compare a string and a number we will get a false value. A similar concept is applied when we use the bang equals like so !=. The big takeway here is always use tripe equals as not checking types can cause unexpected bugs!

Console, Alert and Prompt

Console is an object and we will be using it alot. There are a few methods attached to the console that we will be making use of! We can log a message, error or warning to the console. We can use alert to create a pop up in the browser and give a message to the user. We can also use a prompt to collect user data, although this is not common. Forms are the way to go!

Conditional Statements

If Statements

If statement only runs if the condition is true! The condition only applies to the code inside the if statement! The syntax is like this:if(something is true or false) {do this}

ELSE If Statements

Else If statement has two branchs of logic. Only one branch will run based on if the condition is met! The syntax is like this:if(the condition is met) {do this} else {if the first is false do this instead}

ELSE Statement

Else is the last statement is nothing else is true do this. If anything was true along the way we would not reach the else!

Truthy and Falsey Values!

There are only two values that are true or false, but other values have a truthyness or falsyness to them. This matters for a few reasons. The following values are falsy!

  • false
  • null
  • undefined
  • ""
  • 0
  • NaN

Logical Operators

AND && OR || NOT

The syntax for the AND operator is double ampersand. We would have two expressions and if both sides are true then the whole expression is true! For example we could use this in our password example...if(password.length >= 6 && password.indexOf('') === -1) {console.log("Valid Passord")}. The OR operator behaves differently than and and only cares if one side of the expression evaluates to true! AND has precedence over OR, you can add parentheses to highlight this. The NOT operator inverts the value. If we put it infront of a false value it will reurn false and vice versa
